# 骑手通行证 (Rider Pass) · 完整项目计划书

**版本**: v1.0 | **日期**: 2026-05-14 | **目标**: 交付 Trae 开发原生 App

---

## 一、一句话定位

**全国小区/写字楼入口情报共享平台 — 骑手自己帮自己省时间**

骑手标记哪个门能进、保安什么态度、电梯能不能上——让后来的人少走弯路，多跑几单。

---

## 二、核心痛点（40+条真实声音浓缩）

### 🥇 第一痛：进小区/找楼/等电梯（占33%）

| 子痛 | 真实场景 | 直接影响 |
|------|---------|---------|
| 保安拦路 | 小区不让外卖车进，走路进去一来一回10分钟；甚至有保安围殴骑手事件 | 每单多耗5-15分钟 |
| 找不到楼栋 | 老小区没路牌/单元牌脱落/门牌看不清，晚上/雨天找20分钟 | 高峰期少接2-3单 |
| 电梯难等 | 写字楼排队10多分钟，物业不让上楼只能等顾客下来 | 超时风险暴增 |

### 🥈 第二痛：收入被双重压榨（占23%）

- 单价持续下降：2025→2026从5元降到3元多
- 一个差评扣500元 = 100多单白送
- 平台抽成越来越高

### 🥉 第三痛：超时的锅全是骑手背

- 商家出餐慢 → 骑手超时 → 扣骑手钱（平台政策：商家显示"已出餐"实际没做）
- 客户地址乱写/不接电话 → 白等20分钟
- 系统派单不合理 → 反方向取送

**完整调研报告见附录。**

## 三、为什么做这个？—— 竞品分析

| 竞品/替代方案 | 为什么解决不了 | 骑手通行证的机会 |
|-------------|--------------|----------------|
| 高德/百度地图 | 只有楼栋位置，没有保安态度/入口情报/电梯权限 | 补充「地面情报」维度 |
| 骑手微信群/贴吧 | 信息不可搜索、不可标记地图、消息被刷走 | 结构化+地图化+可检索 |
| 美团官方工具 | 平台无动力让骑手更高效率（怕骑手多接竞品单） | 大厂不做就是蓝海 |
| 饿了么/美团骑手社区 | 只有官方公告，没有用户生成情报 | UGC情报缺口 |

**结论：纯蓝海细分市场，暂无直接竞品。**

## 五、产品设计 — Rider Pass

### 核心功能矩阵

```
P0（必需，不做不上线）
├── Leaflet全屏地图 + 小区标记点（带表情符号区分保安态度）
├── 打卡/录入情报表单（小区名 + 入口情况 + 保安态度 + 备注）
├── 搜索/筛选（按小区名/区域/保安态度）
├── 弹窗详情（入口信息 + 保安态度 + 用户评价）
└── 本地存储（localStorage 持久化，无后端）

P1（重要，Day2做）
├── 贡献排行榜（🥇🥈🥉 激励数据上传）
├── 数据导入/导出 JSON（群内分享）
├── 用户昵称系统（贡献可追溯）
└── 清空重置

P2（锦上添花）
├── 小地图导航跳转（OpenStreetMap/高德跳转）
├── 确认有用/没用 投票
├── GPS自动定位打卡
└── 离线可用（PWA Service Worker）
```

### 已完成的 PWA 前端（可直接供 Trae 参考）

- **地图**: Leaflet + OpenStreetMap tiles + 暗色主题
- **标记**: 保安态度用4种表情符号 😊😐😠🤬 + 4色标记
- **底部导航**: 打卡/列表/排行/我的
- **弹窗表单**: 小区名/入口/保安态度(4选1)/备注
- **排行榜**: 贡献最多的骑手排行
- **数据管理**: JSON导入导出 + 清空

## 八、运营策略

### 定价模型（建议验证阶段）

| 模式 | 说明 |
|------|------|
| 基础版 | 免费 → 查看所有情报、打卡 |
| 情报王 Motive | 排行榜荣誉 + "贡献10条免赞助" |
| 赞助解锁（可选） | 29元/永久 → 导出数据、去广告、VIP标识 |

**核心策略：先用免费版跑通用户，验证后再溢价。**

重点回到第2条策略：贡献10条情报免赞助，用排位赛制驱动冷启动。

### 30天获客路线图

```
Day 1-3:   发到3个骑手群 + 贴吧  → 10-30人
Day 4-7:   知乎/小红书/抖音发体验帖  → 30-100人
Week 2-4:  口碑传播 + 骑手群转发  → 100-500人
```

### 冷启动三板斧

1. **内置100个热门小区基础数据** — 新用户打开就能看到有用内容
2. **骑手群分享码** — 导入导出JSON，群友互传
3. **排行榜激励机制** — 谁贡献多谁上榜，数据王获得群内知名度

## 九、风险评估

| 风险 | 概率 | 影响 | 对策 |
|------|------|------|------|
| 没人传数据 | 中 | 大 | 内置100小区基础数据 |
| 骑手不付费 | 中 | 大 | 免费版有完整核心功能 |
| 平台抄袭 | 低 | 小 | 大厂无动力做，骑手利益驱动 |
| 数据造假 | 中 | 中 | 骑手实名+交叉验证机制 |
| 盈利不足 | 中 | 中 | Day1就开始收赞助，但核心功能免费 |

**放弃标准**：前7天日活 < 10人 → 放弃；30天0赞助 → 全免费化只留住用户。
